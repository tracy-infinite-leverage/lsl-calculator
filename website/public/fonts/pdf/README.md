# `public/fonts/pdf/` — unsubset TTFs for server-side PDF embedding

This folder ships the **unsubset** TrueType source files that the
`@react-pdf/renderer` PDF pipeline (E6.5, `website/src/lib/pdf/`) registers
via `Font.register({ family, fonts: [{ src, fontWeight }] })`. They are
**separate from** the woff2 files in `website/public/fonts/` for two reasons:

1. **Re-subsetting constraint.** The woff2 files in the parent `public/fonts/`
   directory are pre-subset latin-only (~15–18 KB each) — generated by
   `@fontsource/montserrat` / `@fontsource/source-sans-3` for browser delivery.
   `fontkit` (the font engine inside `@react-pdf/renderer`) cannot re-subset
   pre-subset woff2 files; it crashes in `TTFSubset._addGlyph` with
   `RangeError: Offset is outside the bounds of the DataView`.
   Task 5.1's react-pdf spike verified this in May 2026
   (`docs/engineering/changes/2026-05-31-E6.5-task-5.1-react-pdf-spike/HANDOFF.md`
   finding #1). Unsubset TTFs let fontkit perform its own glyph subsetting
   cleanly at PDF render time.

2. **Browser bundle exclusion.** Browser pages load fonts via the `@fontsource`
   packages installed in `package.json`. They never reference this directory.
   The TTFs here are ~3× larger than woff2 and must NEVER ship to the public
   bundle — they live under `public/fonts/pdf/` precisely so we can audit
   exclusion (see `website/scripts/audit-bundle.mjs`).

## Files

| File | Family | Weight | Size | Source |
|---|---|---|---|---|
| `Montserrat-SemiBold.ttf` | Montserrat | 600 | ~444 KB | https://github.com/JulietaUla/Montserrat (official upstream) |
| `SourceSans3-Regular.ttf` | Source Sans 3 | 400 | ~421 KB | https://github.com/adobe-fonts/source-sans (Adobe upstream, `release` branch) |
| `SourceSans3-Semibold.ttf` | Source Sans 3 | 600 | ~416 KB | https://github.com/adobe-fonts/source-sans (Adobe upstream, `release` branch) |

**Note on filename casing:** Adobe ships `Semibold` (lowercase `b`) as the
canonical filename — `SourceSans3-Semibold.ttf`, not `SourceSans3-SemiBold.ttf`.
Honour the upstream casing so updates from the Adobe release branch are a
direct file-replace.

## Licence

All three files are **SIL Open Font License 1.1** — the same licence under
which the matching woff2 files in `public/fonts/` ship. SIL OFL 1.1 explicitly
permits redistribution including embedding inside other documents (e.g. PDF
output of this calculator). No attribution required in the embedded form,
but we credit the upstream foundries in the table above for transparency.

Full licence texts:
- Montserrat — `https://github.com/JulietaUla/Montserrat/blob/master/OFL.txt`
- Source Sans 3 — `https://github.com/adobe-fonts/source-sans/blob/release/LICENSE.md`

## Maintenance

- **Updates:** download fresh canonical files from the URLs above whenever
  the parent `public/fonts/` woff2 family changes — keep weights and family
  versions aligned across the two folders.
- **Adding a weight or style:** add the TTF here, register it in
  `website/src/lib/pdf/fonts.ts`, then bump the relevant Letterhead /
  CitationBlock styling.
- **Adding italic:** none ship today. Task 5.3 (CitationBlock) decided to
  drop italic styling from the citation `note` rather than ship the extra
  TTF. If italic is needed later, both Adobe (`SourceSans3-It.ttf`) and
  Julieta Ula (`Montserrat-Italic.ttf`) provide them under the same OFL 1.1.
